    Project ideas

    Overview

    This project will be a website that provides widespread access to a new decentralised file sharing technology. This decentralised file sharing technology is known as IPFS (InterPlanetary File System). This technology is open-source, meaning it is available to everyone. It provides a service where files are stored on everyone’s computer, compared to centralised servers. This comes with a few unique benefits, such as; privacy, freedom, and lower network capacity requirements which leads to a faster delivery. The type of content that can be delivered, is any type of file. This includes images, videos, PDF’s, Word, as well as website files, and pretty much anything your computer can store.

    Motivation

    The motivation for this project is to enhance my web development skills, learn how to run a business, and create a solution to the lack of consumer access to this new technology. Throughout the development of this website, I will learn many skills in relation to setting up a complex website. It will also have the ability to provide a monetary gain, making it a business forming a foundation for my entrepreneurship. It also provides a solution to consumers who want to access this new technology, without having basic knowledge of this newfound technology.

    Description

    The website, Interplanetary Hub.com is a combination of Google and a forum (like reddit). It allows the sharing of links between users on the website. The interface of the forum is democratic in nature, allowing the content that the user is searching for to be easily accessible. Upon first glance of the website, the user will be presented with a “featured links” section to demonstrate the usefulness of this tool. Under that there will be a search bar where the user can search for the content/file they are looking for. Once the user logs in, they will have the ability to post and vote on links, and perhaps even comment. The downvoting system will help solve the issue of users who post “Bad Links”, as we can monitor a post and its reception, and based on this, act against the post. Bad links could be anything from misleading, phishing, broken, or links malicious intent. The website will guide users into this new technology, known as IPFS. To access content the user will be required to install 2 things; A browser extension, and a desktop application. If the user does not have these installed, the links will not work. The website will be able to detect if a user does not have these installed and create a pop-up window that notifies them of their mistake and shows them how to easily install the IPFS software. This will remove the learning curve of using IPFS, so users can get straight into the content that they want. To monetise the website, there can be multiple sources of revenue. These possible sources are as follow; Advertisements, promoting links, upgraded accounts (i.e. increased functionality), and information can be sold. This income will be used to pay running costs (i.e. Server, Domain registration, Legal Fees, and wages). The website’s functionality will evolve over time, according to new ideas and market trends. This will help the website grow into its ideal form of a platform for everyone to share links to content. The business aspect will also help it to grow as it provides the incentive of money. The more money we get, the more hard work happen, the big more money we get! BIG TIME

    Tools and Technologies

    The languages that the website will be written in is the standard HTML + CSS for the User Interface. On the backend, the website will use PHP and mySQL to retrieve/store data. Python will also be used in order to complete calculations on what should be featured. On the hardware side Google Cloud Service (GCS) will be used to host the website on a LINUX Ubuntu server. The website is also based upon IPFS (Interplanetary File System) Technology which the website at its current state does not actually interact with.
